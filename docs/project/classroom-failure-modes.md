# Classroom Failure Modes

| Failure Mode | What It Looks Like | What To Do |
| --- | --- | --- |
| Incomplete group | One or more students are waiting unexpectedly | Decide whether to wait, skip, or substitute based on the app contract. |
| Broken timing | A timed task expires before students understand it | Stop, restate the rule, and rerun only if the class can absorb it. |
| Wrong headcount | The room has more or fewer participants than expected | Reassign browsers or move to a backup activity. |
| Lost browser | A participant disappears mid-session | Replace the participant, wait briefly, or skip according to the app. |
| Confused payoff | Students cannot interpret the result | Pause and explain the payoff mapping before moving on. |
