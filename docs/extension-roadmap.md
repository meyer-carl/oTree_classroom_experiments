# Extension Roadmap

## First Wave

The first wave has been implemented as a balanced classroom-friendly set:

- `az_endowment_effect`
- `ba_gift_exchange`
- `bb_common_pool_resource`
- `bc_asset_market_bubble`
- loss-aversion extension in `ar_risk_time_preferences`

These additions cover reference dependence, reciprocity, commons problems, and dynamic market mispricing without introducing hard-to-run real-time infrastructure.

## Second Wave

- Discrimination and taste-based/prejudice experiments.
- Voting and median-voter or public-choice exercises.
- Matching and market-design problems.
- Self-control, commitment, and temptation.
- Overconfidence and calibration.
- Ambiguity aversion and loss aversion extensions.

## Delivery Criteria

- One standalone session config per new app.
- One instructor card.
- One debrief note.
- One bot suite.
- One documentation entry in the catalog and roadmap.

## Prioritization Rule

Prefer the app that gives the largest teaching payoff per unit of maintenance cost. If two apps are close, pick the one that is easier to explain in a live class.
