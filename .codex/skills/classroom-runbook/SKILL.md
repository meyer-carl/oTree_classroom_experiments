# Classroom Runbook

Use this skill when documenting how instructors should run oTree sessions.

## Workflow

1. Write the class setup and launch steps first.
2. Document odd headcount and incomplete-group handling.
3. Add timing, debrief, and teardown guidance.
4. Include export and cleanup notes.
5. Keep the language concise enough to use during class.

## Required Topics

- Headcount.
- Timing.
- Setup.
- Expected behavior.
- Failure modes.
- Debrief questions.
- Data export fields.

## Keep In Mind

- Assume the instructor is under time pressure.
- The doc should support a live session, not just a dry run.
